【python开发】— 算法&数据结构（三）

一、基于桶排序思想

1. 计数排序

计数排序是一个非基于比较的排序算法，他的思想是用下标记录每个元素出现的次数，而下标涵盖了所有元素的范围，最后按照下标顺序，次数是多少就输出多少次下标，输出的即为排序后的结果。

1.1 排序过程

注： 该方法存在一个问：无法保证相同元素之间的相对位置。比如上图数字代表年级，同为3年级的有两个班：3年级文科班和3年级理科班，假如排序前文科班排在理科班前面，按上述方法排序完成后无法保证文科班还在理科班前面，因为两个3都无法表示自己是哪个班的。

1.2 排序优化

在上述方法的基础上，计数器 每个下标对应的数都是前面所有数的和 (计数器下标从1开始)，如果计数器下标从0开始，就把下标对应的每个数都减1。接着从要排序的数列的 最后 一个元素开始，在计数器中找对应的下标，例如元素3就找计数器下标3，发现其对应的值为5，则该元素3在排好序的新序列中的位置是下标5的位置，并且计数器下标3对应的数减1；以此类推直到所有元素都放入新列表中，期间一直要保持计数器的更新。

  • 采用该方法排序后的结果可以保证相同元素的相对位置不变，也就是说优化计数排序算法是 稳定 的。
  • 对于把计数器每个下标对应的数都加上前面所有数，可以理解为小于等于2的元素占据着新列表下标从0到3的位置（如下图）；小于等于4的元素占据着新列表下标从0到7的位置。
  • 原列表最后一个3，排序完就在新列表下标为5的位置，然后计数器5位置的值-1变为4，原列表倒数第二个3，排序完就在新列表下标为4的位置。
  • 还有一种情况是元素范围从 非0或非1 开始，比如20-26，那就把新列表建为(26-20+1)的长度。

1.3 代码实现

计数排序代码关键点：计数器、计数器与新列表的对应关系

代码思路：首先得到原列表最大最小值，确定计数器范围；然后遍历原列表，在计数器中记录每个元素出现的次数；再循环将计数器中每个下标对应的数都加上前面所有数，如果计数器下标从0开始，则还需要把计数器中的每个值都 -1；最后对原列表倒序取值，放到新列表的对应位置。

1.4 时间复杂度（O(n+k)）

  • 计数排序不是基于比较的排序算法，因此它的下界可以优于Ω(nlgn)，甚至这个算法都没有出现比较元素的操作。
  • 当n个输入元素的范围是0到k-1或者1到k的整数时，其时间复杂度是O(n+k)。当k=O(n)时，总的时间复杂度为：O(n)，这时采用计数排序就很好，因为这是线性的时间复杂度。
  • 当k=O(n²)或者O(k)>O(n)时，这时计数排序的优势就无法体现，就要考虑是否还要继续用计数排序。因此，计数排序有着限定的适用范围：不仅要排序的数必须是整数，而且还要确保这些整数的范围很小。 例如快速得知高考名次(假设相同分数的人是相同的名次)，高考人数有几十万人，而分数只有0到750分，符合要排序的数是整数且范围很小，可用计数排序。故计数排序很适合对 量大但范围小 的数进行排序。

1.5 空间复杂度（O(n+k)）

计数排序的计数器需要长度为k的空间，长度为n的新列表用于存放排序后的元素，所以计数排序的空间复杂度为：O(n+k)。计数排序也用到了“空间换时间”的思想。

2. 基数排序

  • 基数排序（radix sort）属于“分配式排序”，又称“桶子法”，顾名思义，它是透过键值的部份资讯，将要排序的元素分配至某些“桶”中，藉以达到排序的作用。
  • 基数排序的思想：把待排序的整数按位分，分为个位，十位…..从小到大依次将位数进行排序。实际上分为两个过程：分配和收集。分配就是：从个位开始，按位数从小到大把数据排好，分别放进0–9这10个桶中；收集就是：依次将0-9桶中的数据放进数组中；重复这两个过程直到最高位。

2.1 排序过程

假设有7个十进制的三位数，首先用计数排序对个位数进行排序，然后是十位数，最后排百位数，如果遇到某位数字相同的则不要打乱顺序，按照原顺序排序。个十百三位都排完则排序完成。

2.2 代码实现

基数排序代码关键点：当前在排序的位数、桶

代码思想：找到列表中最大的数(也就是位数最多的)，记录当前在排的是哪一位；初始化桶(0-9)，通过 int((ele / (10**i)) % 10) 就能求出元素的某位属于下标为几的那个桶；最后用两层循环，分别从每个桶里输出元素，排序完成。

2.3 时间复杂度（O(d(n+k))）

对于给定的n个d位数，取值范围为[0,k]，k通常被称为关键字的 基数 。我们使用计数排序比较元素的每一位，计数排序时间复杂度为O(n+k)，需要排序d轮，所以基数排序的时间复杂度为：**O(d(n+k))**。

2.4 空间复杂度（O(n+k)）

基数排序空间复杂度就是在分配元素时，使用的桶空间，其空间复杂度为：O(n+k)

3. 桶排序

桶排序是将待排序集合中处于同一个值域的元素存入同一个桶中，也就是根据元素值特性将集合拆分为多个区域，则拆分后形成的多个桶，从值域上看是处于有序状态的。对每个桶中元素进行排序，则所有桶中元素构成的集合是已排序的。

首先遍历列表找出最小元素与最大元素，根据他们之间的差值选取合适的值域，之后分成若干个桶，把每个元素放到其属于某个值域的桶里，再对每个桶进行桶内排序，最后按顺序输出元素即为排序完成的。

3.2 代码实现

桶排序代码关键点：桶的大小、应该装在哪个桶里

代码思路：桶的大小设定为：最大元素与最小元素的差 / 列表元素的个数；通过 int((i - min_num) // bucket_range) 就可以求得某个元素应该属于下标为几的那个桶里，在桶内排序后回给原列表，排序完成。

3.3 时间复杂度（O(N+N*logN-N*logM)）

第一个循环作用为将待排序集合中元素移动到对应的桶中，复杂度为 O(N)；第二个循环的作用为对每个桶中元素进行排序，并移动回初始集合中，若桶个数为 M，平均每个桶中元素个数为 N/M，则复杂度为 O(N)+O(M*(N/M)*log(N/M)) = O(N+N*(logN-logM)) = **O(N+N*logN-N*logM)**。当 N=M 时，即极限情况下每个桶只有一个数据时，桶排序的最好效率能够达到O(N)。

3.4 空间复杂度（O(N+M)）

桶排序由于需要申请额外的空间来保存元素，并申请额外的数组来存储每个桶，所以空间复杂度为：O(N+M)

4. 计数排序 VS 基数排序 VS 桶排序

这三种排序都是基于桶排序的思想，但对桶的使用上存在差异：

  • 计数排序：每个桶只存储单一键值。
  • 基数排序：根据键值的每位数字来分配桶。
  • 桶排序：每个桶存储一定范围的数值。
  • 计数排序本质上是一种特殊的桶排序，当桶排序中桶的个数取最大( maxV-minV+1 )的时候，就变成了计数排序。
  • 基数排序也是一种桶排序。桶排序是按值区间划分桶，基数排序是按数位来划分；基数排序可以看做是多轮桶排序，每个数位上都进行一轮桶排序。
  • 当用最大值作为基数时，基数排序就退化成了计数排序。
  • 当使用2进制时， k=2 最小，位数 d 最大，时间复杂度 O(d(n+k)) 会变大，空间复杂度 O(n+k) 会变小。当用最大值作为基数时， k=maxV 最大， d=1 最小，此时时间复杂度 O(d(n+k)) 变小，但是空间复杂度 O(n+k) 会急剧增大，此时基数排序退化成了计数排序。
